# 유연근무 출퇴근 기록 앱 기획안

## 1. 배경 및 목적

우리 조직은 **코어타임(10:00–15:00)** 기반 유연근무제를 운영하며, 하루 근무시간은 자유롭게 조절하되 **월 근무시간**을 채우는 방식으로 관리한다. 근무 관리 기준월은 **전월 20일 ~ 당월 19일**이다.

현재 사내 근태 시스템에서 출퇴근 확인이 가능하나, 팀장이 구성원의 실제 출근/퇴근 시각을 파악하기 번거롭다. 이를 보완하기 위해, **구성원이 직접 오늘의 출근/퇴근 시각(변동분)을 간편하게 입력**하고, **팀장이 모바일·웹에서 팀 현황을 한눈에 확인**하며, **월 누적 근무시간이 자동 집계**되는 별도 앱을 만든다.

## 2. 핵심 목표

- **현황 공유**: "오늘 누가 언제 오고, 언제 가는지"를 팀 전체가 실시간 확인
- **근무시간 관리**: 월 기준(20일~익월 19일) 누적 근무시간 자동 집계 및 목표 대비 진척 파악
- **간편 입력**: 출근/퇴근 시각이 평소와 달라졌을 때 몇 번의 탭으로 기록

## 3. 포지셔닝 (기존 시스템과의 관계)

사내 근태 시스템과의 **연동은 보안 정책상 불가**로 확정한다. 따라서 이 앱은 사내 시스템을 대체하는 공식 근태 기록이 아니라, **팀 단위의 현황 공유·자기관리용 독립 도구**로 포지셔닝한다.

- 공식 근태 데이터는 여전히 사내 시스템이 진실 공급원(source of truth)
- 본 앱은 "가벼운 현황판 + 자기 근무시간 트래커"이며, **필요한 정보(출퇴근·부재·연차 등)는 모두 구성원이 앱에 직접 입력**한다
- 연동이 없으므로 **연차·공휴일·출장 등 근무일수에 영향을 주는 정보도 앱 안에서 자체적으로 관리**해야 한다

> 이 포지셔닝 덕분에 법적/규정 부담이 줄고, 빠르게 만들어 실사용을 시작할 수 있다. 다만 연동이 없는 만큼 **입력 편의성과 데이터 정합성**이 성패를 좌우한다.

## 3.5 로그인 · 접근 · 플랫폼 정책 (확정)

실제 구현 시 아래 방침을 확정한다.

**로그인 (첫 화면)**
- 앱 진입 시 **사번(5~6자리 숫자) 입력 로그인**을 첫 화면으로 둔다. 별도 비밀번호 없이 사번으로 본인을 식별(사내 환경 전제, 추후 SSO 확장 가능).
- **명단(구성원 마스터)은 추후 제공**받아 반영한다. 명단은 `사번 · 이름 · 소속 팀 · 근무 지역`을 포함한다.

**팀 기준 근태 공유**
- 로그인하면 **자신이 속한 팀의 구성원 근태를 상호 열람**한다(팀장 전용이 아니라 팀원 모두가 오늘 팀 현황을 봄).
- 목록은 **이름 가나다순** 정렬, **이름 검색/필터**로 특정 인물을 바로 찾는다.
- 타 팀 정보는 노출하지 않는다(프라이버시 §9).

**지역별 기본 출퇴근값 + 변동분만 입력**
- 근무 지역마다 기본 근무시간이 조금씩 다르다: 예) **08:00–17:00** 또는 **08:30–17:30**.
- 오늘 입력 시 **소속 지역의 기본값을 자동 프리필**한다.
  - 평소와 같으면 → **그대로 저장**(확인만)
  - 일찍 왔으면 → **출근 시각만** 수정
  - 늦게 갈 예정이면 → **퇴근 시각만** 수정
- 즉, "바뀐 시간만 바꾸는" 최소 입력을 지향한다.

**플랫폼별 정보 우선순위**
- **모바일**: `오늘의 근태 현황`을 최우선 노출(내 입력 + 우리 팀 오늘 목록). 월간·통계는 후순위/숨김.
- **웹**: 오늘 현황에 더해 `월간 근무시간 현황`·`월간 이력`을 함께 제공.

## 4. 사용자 및 역할

| 역할 | 주요 행동 |
|---|---|
| **구성원(파트원)** | 오늘 출근/퇴근 시각 입력·수정, 본인 월 누적시간 확인 |
| **팀장/파트리더** | 팀 전체 오늘 현황 대시보드 확인, 구성원별 월 누적 진척 확인 |
| **관리자(옵션)** | 팀/구성원 마스터 관리, 근무시간 정책(코어타임·월 목표) 설정 |

## 5. 핵심 기능

### 5.1 출퇴근 입력 (구성원)
- **오늘 출근시각 / 퇴근시각** 입력 — 시간 피커
- **지역 기본값 자동 채움**: 소속 지역 기본값(예 08:00–17:00 / 08:30–17:30)을 프리필 → 변동 없으면 그대로 저장, **바뀐 시간만 수정**(§3.5)
- **근무 형태 태그**: 사무실 / 재택 / 외근
- 하루 중 여러 번 수정 가능(예: 퇴근 예정시각이 바뀌면 갱신)
- 저장 후 **우리 팀 오늘 근태 목록**(가나다순·이름 검색)에서 팀 전체 현황 확인

### 5.2 부재·연차 입력 (구성원) — *신규*
근태 시스템 연동이 없으므로, 근무일수·목표시간에 영향을 주는 부재 정보를 앱에서 직접 관리한다.

- **부재 유형**: 연차(종일) / 반차(오전·오후) / 출장 / 공가 / 병가 등
- **날짜(구간) 지정**: 하루 또는 여러 날 연속 등록
- **목표시간 자동 반영**: 부재 유형에 따라 해당일의 필요 근무시간을 차감
  - 연차(종일) → 그날 목표 근무시간 0h (근무일수에서 제외)
  - 반차 → 그날 목표 4h
  - 출장/공가 → 정책에 따라 근무로 인정(예: 8h 인정) 여부 선택
- **미래 날짜 등록 가능** → 팀 캘린더·현황판에 예정 부재로 표시

### 5.3 예상 퇴근시간 표시
- 출근시각 + 일 근무시간(**8h + 휴게 1h = 재석 9h** 기준)으로 **자동 계산된 예상 퇴근시각** 표시
- 구성원이 직접 예상 퇴근시각을 수동 지정하는 것도 허용
- 코어타임(15시) 이후 퇴근 예정인지 시각적으로 구분

### 5.4 월 목표시간 동적 계산 및 누적 집계 — *핵심*

목표시간은 **고정값이 아니라 기준월마다·구성원마다 다르게 계산**된다.

**목표시간 계산 로직**
```
기준월 근무일수 = (기준월 구간 전체 평일)
                  − (구간 내 공휴일)
                  − (개인 연차/반차 등 부재일 환산)

월 목표시간 = 기준월 근무일수 × 8h
```
- **평일 산정**: 전월 20일 ~ 당월 19일 구간의 월~금
- **공휴일 차감**: 관리자가 등록한 공휴일 캘린더 참조(전 구성원 공통)
- **개인 부재 차감**: 5.2에서 입력한 연차(1일=8h), 반차(0.5일=4h) 등을 개인별로 반영
- 그 결과 **목표시간은 구성원마다 다르다** (예: A는 연차 2일 사용 → 목표 144h, B는 무사용 → 160h)

**누적 집계**
- 기준월 구간의 실제 근무시간 합산
- **진척률 = 누적 근무시간 ÷ 개인 월 목표시간**
- 남은 근무일 대비 **하루 평균 필요 근무시간** 제안: 예) "남은 4일간 하루 평균 7.5h 근무 시 목표 달성"
- 초과/부족 근무 여부 시각화

### 5.5 팀장용 대시보드
- **오늘 현황 보드**: 구성원 카드 목록 — 출근 여부, 출근시각, 예상 퇴근시각, 근무형태(사무실/재택/외근), **부재(연차/반차/출장)**
- **코어타임 준수 현황**: 10~15시 재석 여부 한눈에
- **월 누적 현황**: 구성원별 목표 대비 진척률(진행바) 정렬 — **각자 목표시간이 다름을 전제로 %로 비교**
- **주간 부재 예정**: 이번 주 누가 언제 연차·출장인지 요약
- 모바일·웹 동일 제공

## 6. 화면 구성 (구현 반영)

**0. 로그인** — 사번(5~6자리) 입력 → 본인 식별 (§3.5)

**공통 (팀원 모두)**
1. **오늘 근태** — ⓐ 내 입력 카드(지역 기본값 프리필·변동분만 수정·근무형태) + ⓑ 우리 팀 오늘 근태 목록(가나다순·이름 검색·출퇴근/근무형태/코어타임 재석)
2. **월간 현황** *(웹 우선)* — 팀 월간 진척률(각자 목표 다름, % 비교) / 내 월간 이력(누적·목표·일별 기록)
3. **부재·연차** — 연차/반차/출장 등 등록, 목표시간 반영

**플랫폼별**
- 모바일: `오늘 근태` + `부재·연차`만 노출(월간은 웹)
- 웹: `오늘 근태` + `월간 현황` + `부재·연차` 전체

## 7. 데이터 모델 (개념)

- **User**: id, **사번(로그인 ID, 5~6자리)**, 이름, 소속 팀, 역할, **근무 지역**, 개인 기본 출퇴근시각(지역 기본값 기반)
- **Team**: id, 이름, **근무 지역**(기본 출퇴근값 결정)
- **Attendance(일별 근무 기록)**: user_id, 날짜, 출근시각, 퇴근시각(예정/실제 구분 플래그), 근무형태(사무실/재택/외근), 최종수정시각
- **Absence(부재 기록)**: user_id, 시작일, 종료일, 유형(연차/반차오전/반차오후/출장/공가/병가), 근무인정시간(유형별), 사유(선택)
- **Holiday(공휴일)**: 날짜, 명칭 — 관리자 등록, 전 구성원 공통
- **Policy**: 코어타임(10–15시), 기준월 규칙(20일~익월 19일), 일 근무시간(8h+휴게1h), 부재 유형별 환산 규칙
- **월 목표시간·근무일수는 저장하지 않고, User·Absence·Holiday·Policy로부터 조회 시 동적 계산**(또는 월 마감 시 스냅샷 저장)

## 8. 추가 아이디어 (요청하신 제안)

우선순위를 나눠 제시한다.

**바로 넣으면 좋은 것 (Quick win)**
- **원탭 출근/퇴근 + 위치 무관**: GPS·비콘 없이 "출근했어요/퇴근할게요" 버튼만으로 시작(사내 보안·프라이버시 부담 최소화)
- **오늘의 기본값 제안**: 개인 패턴 학습해 기본값 자동 채움 → 입력 마찰 최소화
- **미입력 리마인더(옵션)**: 코어타임 시작 후 미입력자에게 가벼운 알림(강제 아님)
- **주간/월간 요약 뷰**: 일별 기록을 캘린더 형태로

**팀 협업 강화**
- **간단 코멘트/사유**: 늦은 출근·조기 퇴근 시 한 줄 메모(선택)
- **팀 공유 캘린더 연동(내보내기)**: 앱에 입력된 연차·출장을 팀 캘린더로 내보내 이중 입력 감소(사내 근태 연동과 별개, 캘린더는 보안 부담 적음)
- **회의 시 코어타임 겹침 확인**: 부재 정보를 바탕으로 "이 시간에 A는 외근 중" 안내

**자기관리·인사이트**
- **번아웃 방지 알림**: 월 목표를 크게 초과하는 초과근무 누적 시 본인에게만 알림
- **월말 마감 리포트**: 기준월 종료 시 본인 요약(총 근무시간, 목표 달성률, 평균 출퇴근시각)
- **패턴 인사이트**: "이번 달 평균 출근 9:40, 지난달 대비 20분 빨라짐" 등

**운영/신뢰성**
- **수정 이력(revision) 관리**: 출퇴근 시각을 나중에 고치면 변경 로그를 남겨 신뢰성 확보(사내 DANDI식 revision 히스토리 패턴 차용 가능)
- **입력 마감 규칙**: 당일 자정 이후 수정 제한 또는 "사후 수정" 표시로 데이터 신뢰도 관리
- **프라이버시 원칙 명문화**: 위치 추적 없음, 팀장 열람 범위 한정, 자기관리 목적 명시 → 구성원 수용성 확보

**향후 확장(우선순위 낮음)**
- 사내 시스템 단방향 동기화(보안 검토 통과 시)
- 유연근무 통계 대시보드(실 단위)
- 회의/코어타임 겹침 자동 감지

## 9. 프라이버시·수용성 고려사항 (중요)

출퇴근 기록은 민감한 데이터이므로 설계 초기부터 다음을 원칙으로 한다.

- **위치·감시 요소 배제**: GPS, 자동 추적 없이 자발적 입력 기반 → 감시 아닌 자기관리 도구로 인식되게
- **열람 범위 제한**: 팀장은 소속 팀 구성원만, 구성원 간 상호 열람 범위는 정책으로 조정
- **강제성 최소화**: 리마인더는 알림 수준, 미입력에 대한 페널티성 기능 지양
- **투명성**: 누가 내 기록을 볼 수 있는지 앱 내 명시

## 10. 단계별 로드맵 (제안)

- **1단계 (MVP)**: 구성원 출퇴근 입력 + 부재/연차 입력 + 예상 퇴근시간 + 팀장 오늘 현황 보드
- **2단계**: 공휴일·부재 반영한 **월 목표시간 동적 계산** + 누적 진척률 + 월말 리포트
- **3단계**: 팀 캘린더 내보내기, 수정 이력 관리, 인사이트
- **4단계 (선택)**: 연차 잔여일수 관리, 승인 워크플로우

## 11. 열린 질문 (기획 확정을 위해 논의 필요)

- **출장·공가·병가의 근무 인정 기준**: 각각 몇 시간(예: 8h)을 근무로 인정하는가? 목표시간에서 빼는가, 근무한 것으로 채우는가?
- **공휴일 캘린더 관리 주체**: 관리자가 매년 등록하는 방식으로 충분한가?
- **연차 잔여일수 관리까지 할 것인가**, 아니면 "부재일 등록"만 하고 잔여 관리는 범위 밖인가?
- 팀장 외 구성원끼리도 서로의 출퇴근·부재를 보게 할 것인가?
- 부재 입력의 신뢰성: 승인 절차(팀장 확인) 없이 자기 신고만으로 목표시간을 조정해도 되는가?
